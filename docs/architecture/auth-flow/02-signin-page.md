# Сторінка авторизації

Файл: `apps/web/src/app/[locale]/auth/signin/page.tsx`

## UI елементи

```
┌─────────────────────────────────────────┐
│                                         │
│            Вхід до CyanShip            │
│            (підзаголовок)               │
│                                         │
│  ┌───────────────────────────────────┐  │
│  │  Email                            │  │
│  └───────────────────────────────────┘  │
│  ┌───────────────────────────────────┐  │
│  │           Продовжити              │  │
│  └───────────────────────────────────┘  │
│                                         │
│           ──── або ────                 │
│                                         │
│  ┌───────────────────────────────────┐  │
│  │      Увійти з Google              │  │
│  └───────────────────────────────────┘  │
│                                         │
└─────────────────────────────────────────┘
```

## Поведінка

1. Юзер вводить email і натискає "Продовжити"
2. Показується прелоадер (стан `loading`)
3. Backend `POST /api/auth/check-email` перевіряє: чи існує юзер і чи має пароль
4. В залежності від результату — один з трьох сценаріїв (див. [Сценарії email-авторизації](./03-email-scenarios.md))

## State machine

Сторінка реалізована як state machine з 6 станами:

| Стан | Опис |
|------|------|
| `email` | Початковий — поле email + Google кнопка |
| `loading` | Прелоадер під час check-email |
| `password` | Поле пароля (Сценарій A) |
| `magic-link-sent` | Повідомлення "Перевірте пошту" |
| `recovery` | Відновлення деактивованого акаунту |
| `error` | Помилка з кнопкою повернення |

> **Безпека:** Endpoint `check-email` захищається rate limit per-IP (10 запитів за 60 секунд). Ключ: `check_email:{ip}`.

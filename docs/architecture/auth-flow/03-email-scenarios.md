# Сценарії email-авторизації

## Сценарій A: Юзер існує + має пароль

**Тригер:** Backend повертає `{ hasPassword: true }`

**UI:** Під полем email з'являється поле пароля (з toggle show/hide) + кнопка "Увійти" + посилання "Забув пароль?" + кнопка "Змінити" для повернення до кроку вводу email.

```
┌─────────────────────────────────────────┐
│  ┌───────────────────────────────────┐  │
│  │  email@example.com     [Змінити] │  │
│  └───────────────────────────────────┘  │
│  ┌───────────────────────────────────┐  │
│  │  Пароль                      [*] │  │
│  └───────────────────────────────────┘  │
│              Забув пароль?              │
│  ┌───────────────────────────────────┐  │
│  │             Увійти                │  │
│  └───────────────────────────────────┘  │
└─────────────────────────────────────────┘
```

- **Show/hide toggle:** Іконка у полі пароля перемикає видимість.
- **"Змінити":** Повертає на крок вводу email (state `email`), очищає пароль і помилки.

**Після успішного входу:** `getMe()` -> оновлення Zustand store -> redirect на `/profile`.

---

## Сценарій B: Юзер існує + НЕ має пароля

**Тригер:** Backend повертає `{ hasPassword: false, isNewUser: false }`

**UI:** Автоматично відправляється magic link з purpose `login`. Прелоадер зникає, стан переходить в `magic-link-sent`:

```
┌─────────────────────────────────────────┐
│                                         │
│        Перевірте вашу пошту             │
│                                         │
│  Ми надіслали посилання для входу на    │
│  email@example.com                      │
│                                         │
│        <- Інший email                   │
│                                         │
└─────────────────────────────────────────┘
```

**Email:** Шаблон LOGIN — "Натисніть кнопку нижче, щоб увійти у ваш акаунт."

---

## Сценарій C: Юзер НЕ існує (новий)

**Тригер:** Backend повертає `{ hasPassword: false, isNewUser: true }`

**UI:** Ідентичний сценарію B. Автоматично відправляється magic link з purpose `register`.

**Email:** Шаблон REGISTER — "Натисніть кнопку нижче, щоб завершити реєстрацію."

> **Важливо:** З точки зору UI, сценарії B та C виглядають однаково для юзера. Різниця тільки в purpose magic link та тексті email.

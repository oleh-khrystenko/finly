# CyanShip — Опис проекту

## Що це

CyanShip — це production-ready SaaS-бойлерплейт на базі Next.js (App Router) + NestJS, організований як Turborepo-монорепозиторій. Він надає повністю реалізоване ядро для швидкого запуску web-додатків: авторизація (Google OAuth, Magic Link, Password), платіжна система (Stripe subscriptions + one-off credit packs), i18n (uk/en), theming (light/dark/system), profile management, soft-delete з grace period — все з коробки, з E2E-тестами та strict TypeScript.

CyanShip одночасно виконує дві ролі:

1. **Бойлерплейт для клієнтських проектів.** При старті нового проекту робиться форк репозиторію, видаляється модуль agency (бізнес-логіка агенції), і розробка клієнтського продукту починається поверх готового, відтестованого ядра. Це скорочує шлях від ідеї до робочого MVP з місяців до 3-4 тижнів.

2. **Живий лендінг та вітрина агенції.** Сам сайт CyanShip — це діючий додаток, де потенційний клієнт може зареєструватися, протестувати Stripe Checkout, побачити якість UI/UX та переконатися, що його майбутній продукт буде побудований на надійній основі.

## Архітектурне розділення Core / Agency

Ключова архітектурна особливість — жорстке розмежування між двома шарами:

- **Core** — авторизація, користувачі, платежі, shared UI, валідація, i18n. Це стабільне ядро, яке повторно використовується в кожному проекті. Core не залежить від agency і не знає про його існування.

- **Agency** — бізнес-логіка агенції: лендінг, лід-магніти, демо-функціональність. Цей модуль ізольований від core на рівні файлової структури, ESLint-правил та окремого entry point в packages/types. Видаляється за 15 хвилин при форку для клієнта.

Одностороння залежність (Agency -> Core, ніколи навпаки) гарантує, що клієнтський код ніколи не залежить від чужої бізнес-логіки, а core залишається чистим та переносимим.

## Для кого це

CyanShip створений для обслуговування двох категорій клієнтів:

- **Нетехнічні фаундери стартапів** (pre-seed / seed стадія), яким потрібен робочий прототип для демонстрації інвесторам — не мокапи, а реальний додаток з auth і payments. Вони цінують швидкий Time-to-Market і хочуть отримати результат без занурення в технічні деталі.

- **B2B-бізнеси**, які шукають кастомну платформу або внутрішній інструмент з передбачуваними термінами та бюджетом.

Цільова географія: США, Велика Британія, Західна Європа, Скандинавія.

## Бізнес-модель

Послуга — запуск SaaS MVP та кастомних B2B-платформ "під ключ" за 3-4 тижні.

Ціноутворення: від $1 500–$2 000 на початковому етапі (напрацювання портфоліо), з подальшим зростанням до $3 000–$5 000+ за проект.

Рутинний код пишеться за допомогою LLM (Claude Opus), але архітектурні межі, типізація та зв'язки в монорепозиторії контролюються вручну — це дозволяє масштабувати швидкість без втрати якості.

## Канали залучення клієнтів

- **LinkedIn** — холодний аутріч фаундерів стартапів.
- **X / Twitter** — Build in Public, моніторинг болей ЦА (скарги на повільну розробку, зірвані дедлайни).
- **Wellfound** — стартапи на стадії pre-seed/seed, контрактна робота.
- **Лендінг CyanShip** — живе демо як головний інструмент конверсії.
- **AI Scope Generator** (в розробці) — лід-магніт, який перетворює хаотичні ідеї клієнта на технічний план та естімейт, дозволяючи приходити до ліда з готовим рішенням.
- **Соціальний доказ** — демонстрація пет-проекту Wish Hub через Loom-відео (акцент на UI/UX). Код клієнтів ніколи не показується (NDA).

## Конкурентні переваги

- **Швидкість** — 3-4 тижні замість місяців, бо core вже готовий і покритий тестами.
- **Прозорість** — клієнт бачить живе демо ядра ще до початку роботи.
- **Модульність** — чітке розділення core/agency гарантує чистоту клієнтського коду.
- **Enterprise-якість з коробки** — TypeScript strict, Zod-валідація, E2E-тести, two-phase webhook idempotency, brute force protection, token rotation з reuse detection.
